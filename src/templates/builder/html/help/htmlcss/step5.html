Now we have a nice homepage. But what about the other pages? In order to not copy & paste everything or even create everything anew, Prails has a concept, called decorators. This means, an event handler can decorate itself with the result of another event handler. This decoration usually is something like the header and footer, sometimes additional navigation elements, whereas the content remains at the actual event handler that was called. Now, in order to prepare for creating new pages that all share our header including the navigation menu and the footer, we'll create such a decorator.

Therefore, go back to the list of event handlers at the Bakery module. Then click the Add button and enter a name for the decorator into the event handler name's field: decorate. Then move all the output code from the home event handler over to the new event handler. But keep everything inside the SECTION tag in the home handler's output code.

In the decorate event handler, now remove everything inside the SECTION tag and instead write there: <c:body/>. This by itself declares the event handler as a decorator. Save it and go back to the home event handler.

Here we still have to use it. For this, make the output code column smaller again, by pressing the - button right of it. Once the Handler code column is visible again, change it's last line to <- out param, "Bakery:decorate", which will tell the home event handler when run, that it should use the decorate event handler from the Bakery module for it's decoration.
