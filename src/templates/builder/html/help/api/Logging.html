Logging

For logging functionality there is a class called Logger. An instance of this class is always in the global context, which let's you write log messages into the framework.log file within the log directory. If you need some special purpose logs, you can create your own instances of that class and construct it with the path to the log file you want. By setting the global configuration value DEBUG_LEVEL to one of the following values, you can define what should be put into the logs and what should not.

Log Level Log Messages written
0fatal
1warn, error, fatal
2debug, warn, error, fatal
3info, debug, warn, error, fatal
4trace, info, debug, warn, error, fatal

Log level 2 is the default. This should be used in production environments.

As the log messages above indicate, the respective methods that can be used are:

fatal($str_msg, $bol_strace = false) error($str_msg, $bol_strace = false) warn($str_msg, $bol_strace = false) debug($str_msg, $bol_strace = false) info($str_msg, $bol_strace = false) trace($str_msg, $bol_strace = false)

The first parameter represents the message to be written into the error log, the second parameter is optional. If set to true it will additionally write the current stack trace into the error log. Default is false. fatal log messages will additionally throw an exception.

Example:

// first getting the global logger global $log; $log->debug("This is just a test"); // creating a custom logger $myLog = new Logger("log/mycustom.log"); $myLog->error("Let's see how an error looks like...");
