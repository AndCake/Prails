TagLibs

A TagLib is a library of special tags, that will be replaced during the include process of a template, to be precise as part of the template's compilation, by their actual content. The tag name is identified by it's file name. So if the file is called box.tag, the respective tag, that would then be usable, would be called . All special tags are prefixed with the "c" prefix (so it's actually , for example). Any valid PHP code that is part of the tag lib definition, will be evaluated at compile time. Special tags can have attributes and content. The attributes can be accessed in the tag file by $tag["attributes"][""]. The tag's content can be accessed in the tag file by $tag["body"]. This code is written in normal PHP code.

There also exists a special notation for PHP code, that is to be generated during compile time by the tag: <% [php code goes here] %> . Furthermore the compiler environment for tags also has the possibility to convert macro-style variables to PHP variables at include time. This macro-style always looks like #namepart.subpart, whereas there can be multiple subparts be following after the first one (but may not). The compiler converts these expressions actually into code that looks like (with all of it's subparts, of course).

As part of the standard TagLib, some tags are already predefined: foreach, if and include (more to come).
